# AI-oral-exam

AI-oral-exam 是一个面向课程答辩、学习成果核验与大规模口试场景的开源 AI 口试系统。项目希望用实时语音交互、多智能体协作和可配置评分规则，帮助教师完成第一轮高并发、标准化的口试筛查，再把更需要人工判断的学生表现交给教师进行二次复核。

项目仓库：[https://github.com/xyp-carry/AI-oral-exam](https://github.com/xyp-carry/AI-oral-exam)

## 项目背景

生成式 AI 改变了学生获取知识和完成作业的方式，也让传统作业与笔试更难单独证明学习成果。对于教师而言，面对几十到上百名学生时，逐一组织人工答辩会消耗大量时间；但如果完全不做口头追问，又很难判断学生是否真正理解自己的作业、代码或报告。

AI-oral-exam 试图解决这个矛盾：让 AI 先承担标准化、可重复、可规模化的基础口试流程，围绕学生提交的材料进行追问、记录、评分和初筛；教师再根据系统结果聚焦高风险或高价值样本，完成更深入的人工评估。

## 系统定位

本项目不是简单的聊天机器人，而是一个围绕“口试流程”组织的教学评估系统。它关注四件事：

- 让学生能够通过语音完成自然的一问一答。
- 让 AI 能够读取考试规则、学生材料和教师提供的评分标准。
- 让多个智能体分别承担提问、追问、评分、文件解析与工具调用等角色。
- 让教师能够管理身份、会话、资源与考试结果。

## 核心模块

### 人机交互模块

人机交互模块负责打通实时语音闭环，使学生可以通过语音完成考试。系统预期支持 ASR 语音识别、LLM 回答生成、TTS 语音播报，并通过 WebRTC 建立实时交互链路。

### 数据整理模块

数据整理模块负责把“考试要求”和“学生材料”转化为可被 AI 使用的上下文。系统支持上传考试规则文件，并支持解析学生的个性化资料，例如图片、文本报告、代码或文档，从而生成更贴近个人作业内容的针对性问题。

### 多智能体协作模块

多智能体协作模块模拟真实考委会的分工。ExamSetterAgent 负责出题，Interviewer 负责与学生对话并持续追问，JudgerAgent 负责对学生回答进行多维评分，FileParser 负责解析材料，相关工具模块用于代码读取、文档解析、仓库下载和检索增强。

### 口试管理系统

口试管理系统负责组织考试规则、候选人状态、出题、面试、判卷和教师复核流程。它与会话系统、工具系统和基础监控协作，使项目从单次语音问答扩展为完整的教学口试平台。

## 流程逻辑

系统以学生和教师两类用户为核心。学生通过通信模块参与口试，通信模块负责 TTS 播报、STT 识别和出题交互；教师负责配置判定模块、出题模块和专业知识库，为口试提供规则与数据支撑。

学生上传仓库或压缩包，系统从中解析报告与代码。报告进入 RAG 模块形成专业数据支撑，代码进入 Codeanalysis 模块辅助查看与分析。出题模块结合教师配置、问题池、RAG 检索结果和代码分析结果生成问题，再通过通信模块发送给学生。

判定模块由阶段判官、总判官和报告判官组成：阶段判官关注当前回答是否达标，总判官负责整体表现判断，报告判官结合上传材料和过程记录给出报告层面的评价。最终形成“上传材料 - 生成问题 - 语音问答 - 多维判定 - 教师复核”的闭环。
