# 项目计划

项目计划按“先验证语音链路，再扩展智能体与工具，最后形成可运行单机版”的路径推进。下面内容根据现有项目报告整理。

## 阶段目标

| 时间 | 目标 | 产出 |
| --- | --- | --- |
| 2026.3.30 - 2026.4.26 | 调研并规划项目架构，实现初版语音交互功能模块，验证方案可行性。 | 初始系统架构与项目计划文档 |
| 2026.4.27 - 2026.5.2 | 搭建初版 Agent 体系，实现工具调用与测评代码。 | Agent 与 Tool 代码模块 |
| 2026.5.3 - 2026.5.10 | 实现多模态文档上传解析工具，参考 Agent 工具 harness 实现提问功能。 | 文件解析与提问能力 |
| 2026.5.11 - 2026.5.17 | 实现 Interviewer、FileParser、Judge、Coder，完成后端初版功能。 | 多智能体后端雏形 |
| 2026.5.18 - 2026.5.24 | 实现资源 info 日志生成、Client 角色认证和消息接收模块。 | 资源调度与管理基础 |
| 2026.5.25 - 2026.5.31 | 结合语音交互系统，实现 AI 口试单机版初版。 | 可演示的单机版系统 |

## 近期关注点

- 稳定实时语音链路，保证 ASR、LLM、TTS 之间的延迟可接受。
- 明确考试规则的数据结构，避免评分标准只以自然语言散落在提示词中。
- 打通学生材料解析到个性化追问的链路。
- 让 Judge 输出可解释的评分结果，便于教师复核。
- 为会话状态、日志和资源消耗建立统一记录。

## 后续可扩展方向

- 引入更细粒度的评分 rubrics，支持不同课程或题型复用。
- 增加多评委评分一致性检查，降低单模型偏差。
- 完善教师端批量管理、异常会话重试和人工复核流程。
- 支持更多文件类型和代码语言的解析与验证。
