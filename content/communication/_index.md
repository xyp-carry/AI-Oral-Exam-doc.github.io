# 通信层

通信层直接使用 Pipecat 作为实时通信手段。本质上，通信层通过 WebRTC 建立语音通信连接，并通过 Pipeline 组织一条可插拔式处理链路，使 STT、LLM 上下文处理、TTS 等能力可以按需要替换或扩展。

## WebRTC

WebRTC 负责承载实时语音通信，让学生和系统之间可以进行低延迟的语音交互。通信层不直接把语音问答写死在业务逻辑中，而是把实时音频输入和输出交给 Pipecat 管理。

## Pipeline

Pipeline 是通信层中的可插拔式链路。一次口试中，音频输入、语音识别、上下文处理、文本输出和语音合成都可以被组织到 Pipeline 中。后续如果需要替换 STT、TTS 或增加新的中间处理步骤，可以通过调整 Pipeline 完成。

## STT

STT 使用阿里的 SenceVoice，将学生语音先转换为文字。完成语音转文字后，系统会把当前上下文和转换后的文字一起送入后续 LLM 处理流程，用于进行专用语纠正，减少课程术语、项目术语或专业词汇被识别错误带来的影响。

![STT 流程](../img/stt-sencevoice-flow.png)

图中 STT 负责完成基础语音识别，LLM 结合 external info 对识别结果进行上下文修正，使最终文本更贴近当前课程、考试和项目语境。

## TTS

TTS 负责把系统生成的文本转换为语音并播报给学生。当前设计中，TTS 直接调用 Minimax 的 API 接口完成语音合成。
