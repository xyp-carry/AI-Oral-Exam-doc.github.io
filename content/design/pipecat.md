# 实时语音链路 Pipecat

实时语音链路负责把一次口试组织成连续的语音会话。项目采用可插拔的 Pipecat pipeline，把输入音频、语音识别、LLM 推理、语音合成和输出音频串联起来，并通过 WebRTC 支撑低延迟交互。

## 核心流程

1. 学生通过浏览器或客户端进入口试会话。
2. 会话系统接收学生语音流，并交给 STT 模块转写为文本。
3. 口试上下文、历史回答、考试规则和学生材料被传入 LLM。
4. LLM 输出问题、追问、提示或阶段性反馈。
5. TTS 将文本结果合成为语音并播报给学生。
6. 会话系统持续记录问答过程，为后续判卷和教师复核提供依据。

## 可插拔 pipeline

会话系统支持多个全双工对话可插拔 pipeline。每条 pipeline 可以按考试场景切换不同的 STT、LLM、TTS 或中间处理组件，例如：

- 面向低延迟口试的实时语音 pipeline。
- 面向长答案分析的文本增强 pipeline。
- 面向特殊课程的自定义追问 pipeline。

这种设计让系统能够根据课程、考试规模和模型服务能力逐步扩展，而不是把语音能力固定在单一实现上。

## 与口试管理系统的关系

Pipecat 链路只负责实时对话执行，口试管理系统负责决定“问什么、如何评分、何时结束”。两者通过会话状态和上下文交换协作：会话系统提供语音与转写结果，口试管理系统提供题目、追问策略和评分任务。
